# CLAUDE.md — правила сборки макетов

Краткие обязательные правила для прототипов этого репозитория. Полные технические
заметки и грабли по DS-компонентам — в `SESSION-INSIGHTS.md`; здесь только то, что
ломается чаще всего.

**Спеки разделов** (контракты для сборки контента агентом):
- `docs/feeds-spec.md` — Activity-лента и Q3-лента (типы карточек, поля, колонки
  листа, табы, реген).

---

## 0. DS-компоненты и DS-токены — без хардкода

Базовое правило для всех макетов:

- **Компоненты — из DS.** Есть готовый компонент в `components/*.css` (uni-card,
  uni-cell, button-inline/button-container, button-circle, avatar, icon, media,
  header, nav-bar, tabbar, chip, tooltip …) — используй ЕГО, а не верстай свой
  аналог руками. Нового компонента нет в DS → только тогда делай кастом.
- **Значения — из токенов** (`tokens.css`). Цвета/фоны/отступы/радиусы/типографика
  через `var(--…)` или `ds-*`-классы. ❌ Никакого хардкода дизайн-значений:
  - цвет/фон литералом (`#fff`, `rgba(...)`, `white`) → `var(--dynamic-…/--static-…)`;
  - отступ «магическим» px (`12px`, `16px`, `8px`) → `var(--space-3/4/2)` (сетка 4px);
  - размер шрифта литералом (`17px`) → `ds-*`-класс (`ds-title-m`, `ds-body-m`).
  - Карта токенов — `tokens.css` + таблица в `SESSION-INSIGHTS.md`.

  Исключения (не хардкод DS): aspect-ratio, `width:100%`, `0`, `1px`-бордер, URL
  картинок, уникальные позиционные сдвиги оверлеев.

Это правило проверяет субагент `macket-rules-checker` (см. §3).

## 1. Кнопка «Назад» в навбаре

**Всегда** используй DS-иконку `__slot-back`, а не `<img>` с хардкодом файла.

```html
<span class="button-inline-wrapper __size-24 __view-secondary">
  <button class="button-inline __size-24 nav-bar__back" data-href="<куда>" aria-label="Назад">
    <span class="button-inline__content">
      <span class="icon __size-24 __slot-back"></span>
    </span>
  </button>
</span>
```

- `.icon.__slot-back` уже замаплен в `components/icon.css` на `arrow_left_24.svg` —
  это **стрелка влево `←`**, правильный дефолт.
- ❌ Не вставляй `<img src="assets/icons/back_24.svg">` — это **шеврон `‹`**, не стрелка.
  Именно из-за такого хардкода кнопка «назад» регулярно оказывается неправильной.
- Класс `nav-bar__back` подключает централизованный обработчик из
  `components/screen-transition.js` (анимация «назад» + история). Не вешай свой
  `history.back()` на отдельный `id`, если можно использовать `nav-bar__back`.
- В New Vision (`/new-vision/` или `?nv=1`) `__slot-back` переопределяется на
  NV-глиф через `components/nv-navbar.css` — это норма, отдельно ничего не делай.

## 2. Таб-бар в подпапочных прототипах (`activity-lenta/` и подобных)

Страницы в подпапке имеют `<base href="../">` — **все** относительные пути
(включая `location.href` из `tab-bar.js`) резолвятся в **корень репозитория**.

Поверх этого работает `components/proto-contain.js` — «контейнер», который гасит
любую навигацию за пределы папки текущей страницы. Поэтому маршруты таб-бара
(`messages.html`, `klipy.html`, `menu.html`, `tribune.html`, `lenta-q3.html`)
по умолчанию считаются «чужими» и **блокируются** → таб-бар «не работает».

Правила:

- Слоты, которым **разрешён** выход в общие разделы приложения, перечислены в
  `TAB_ALLOW_OUT` внутри `proto-contain.js`. Сейчас это `message`, `clip`, `menu`.
  Нужен ещё слот (например `book` → `tribune.html`) — **добавь его туда**, иначе он
  будет молча заблокирован.
- Сам маршрут слота берётся из `components/tab-bar.js` → `ROUTES`
  (`feed → lenta-q3.html`, `book → tribune.html`, `message → messages.html`,
  `clip → klipy.html`, `menu → menu.html`) или из `data-href` на кнопке.
- `feed` обычно `__state-on` (текущая вкладка) и навигации не делает — это норма.
- После правок таб-бара **проверяй в браузере**: на первом заходе
  `activity-lenta/lenta.html` штатно редиректит на онбординг-шторку
  `add-friends-sheet.html` (гейт `afs-seen-al` в `sessionStorage`). Чтобы тестировать
  сам таб-бар, заранее выставь `sessionStorage.setItem('afs-seen-al','1')`.

Чеклист «таб-бар не навигирует»:
1. Страница в подпапке? → `proto-contain.js` блокирует — проверь `TAB_ALLOW_OUT`.
2. У слота есть маршрут в `tab-bar.js ROUTES` или `data-href`?
3. Не словил ли редирект на онбординг (см. выше)?

## 3. Проверка перед коммитом

**Перед коммитом правок макетов — авто-ревью субагентом `macket-rules-checker`**
(`.claude/agents/macket-rules-checker.md`). Он grep-first сверяет дифф с правилами
этого файла и `docs/feeds-spec.md` и возвращает вердикт **PASS / FAIL**.

- Гоняй его **один раз — когда правки готовы к коммиту** (не на каждой микроитерации:
  это субагент, лишние прогоны жгут токены/время). По ходу работы для быстрой саморверки
  достаточно самому прогрепать пару признаков.
- **Коммить только при `PASS`.** `FAIL` → почини по отчёту и прогони ещё раз.
- `WARN` (напр. «интерактив не проверен в браузере») не блокирует, но разберись.
- Чек-лист агента сопровождает не он сам, а `macket-insights-curator` (раз в неделю по
  крону): переносит устойчивые находки из инсайтов/шины в правила.

Интерактивные правки (таб-бар, навигация, кнопка «назад») сверх этого **проверяй
реальным прогоном в браузере** (Playwright + Chromium уже есть в окружении; или
субагент `screenshot-testing`), а не на глаз. Минимум — кликни вкладку/кнопку и
убедись, что URL меняется на ожидаемый.
