# Переезд: код → GitVerse, хостинг → Yandex Object Storage

Прототип — **полностью статический** сайт (HTML/CSS/JSON/шрифты), бэкенда нет.
Поэтому «развернуть хостинг» = отдать файлы по HTTP из бакета. Ниже — полный
маршрут переезда.

---

## Часть 1. Перенос кода в GitVerse

GitVerse — российский git-хостинг (СберТех), ближайший аналог GitHub.

1. Зарегистрируйтесь на <https://gitverse.ru> (вход по Сбер ID / почте).
2. Создайте пустой репозиторий, например `ds-design` (без README, чтобы не
   конфликтовал с историей).
3. В локальной копии добавьте новый remote и запушьте всё:

   ```bash
   git remote add gitverse https://gitverse.ru/<ваш-логин>/ds-design.git
   git push gitverse --all
   git push gitverse --tags
   ```

4. Для приватного репозитория GitVerse попросит логин/пароль или
   персональный токен (Settings → Access tokens). Удобнее настроить SSH-ключ
   (Settings → SSH keys) и использовать `git@gitverse.ru:...`.
5. Когда убедитесь, что всё на месте, можно сделать GitVerse основным:

   ```bash
   git remote rename origin github-old   # на всякий случай
   git remote rename gitverse origin
   ```

## Часть 2. Хостинг прототипа в Yandex Object Storage

### 2.1. Аккаунт и каталог
1. Зайдите в <https://console.yandex.cloud>, создайте облако и каталог
   (folder). Привяжите платёжный аккаунт (для такого объёма — копейки в месяц).

### 2.2. Сервисный аккаунт и ключ доступа
1. В каталоге: **Сервисные аккаунты → Создать**, роль `storage.editor`.
2. Откройте созданный аккаунт → **Создать новый ключ → Статический ключ
   доступа**. Сохраните `Идентификатор ключа` и `Секретный ключ` — это
   `AWS_ACCESS_KEY_ID` и `AWS_SECRET_ACCESS_KEY`.

### 2.3. Бакет
1. **Object Storage → Создать бакет**. Имя должно быть глобально уникальным,
   например `ok-ds-prototype` (только латиница/цифры/дефис).
2. Тип доступа на чтение объектов — **Публичный**.
3. Класс хранилища — Standard.

### 2.4. Включаем хостинг сайта
1. В бакете откройте вкладку **Веб-сайт → Хостинг**.
2. **Главная страница (index document):** `index.html`
   (он уже есть в репозитории и редиректит на `preview.html`).
3. Сохраните. Адрес будет вида:
   `https://<имя-бакета>.website.yandexcloud.net/`

### 2.5. Заливка файлов

Вариант А — мышкой: на вкладке **Объекты** перетащите все файлы проекта
(кроме `.git`, `.github`, `*.md`, `*.sh`). Быстро для разового деплоя.

Вариант 0 — одной командой (`setup-yandex.sh`, проще всего): после шагов 2.2–2.3
(сервисный аккаунт + статический ключ) можно не настраивать бакет руками, а
запустить скрипт — он сам создаст бакет, включит хостинг и зальёт сайт:

```bash
pip install awscli
export AWS_ACCESS_KEY_ID=<идентификатор ключа>
export AWS_SECRET_ACCESS_KEY=<секретный ключ>
export BUCKET=ok-ds-prototype
./setup-yandex.sh
```

Вариант Б — пошагово, скриптом заливки (повторяемо):

```bash
# Поставьте AWS CLI (Object Storage S3-совместим):
pip install awscli            # или: brew install awscli

export AWS_ACCESS_KEY_ID=<идентификатор ключа>
export AWS_SECRET_ACCESS_KEY=<секретный ключ>
export BUCKET=ok-ds-prototype

./deploy.sh
```

`deploy.sh` лежит в корне репозитория и делает `aws s3 sync` с нужными
исключениями.

### 2.6. HTTPS и красивый домен (опционально)
- Адрес `*.website.yandexcloud.net` работает по HTTP. Для HTTPS и/или своего
  домена подключите **Yandex Cloud CDN** перед бакетом и выпустите сертификат
  в **Certificate Manager** (бесплатный Let's Encrypt). Домен заводится через
  **Cloud DNS**.
- Для прототипа «на завтра» HTTP-адреса бакета обычно достаточно.

## Часть 3. Автодеплой из GitVerse (опционально)

Чтобы каждый пуш в `main` сам раскатывал прототип, в репозитории лежит
`.gitverse/workflows/deploy.yml`. В настройках репозитория GitVerse заведите
секреты `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `BUCKET` — и деплой
будет автоматическим. Синтаксис GitVerse CI совместим с GitHub Actions; при
расхождениях сверяйтесь с docs.gitverse.ru.

- [ ] Хостинг сайта включён, index document = `index.html`
- [ ] Файлы залиты (`./deploy.sh` или мышкой)
- [ ] Прототип открывается по `https://<бакет>.website.yandexcloud.net/`
