# Сборка «Вокруг вас» из Google-таблицы

Источник: таблица, лист **«Вокруг нас»**.
Spreadsheet ID: `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`

Колонки листа (читаются **по названиям**, порядок столбцов любой):

`лид · кто · бейдж · текст · кнопка · категория · изображение` (+ опц. `id`)

Колонки ищутся по ключевому слову (можно с уточнениями, напр. «изображение (если
есть)»). `id` необязателен — если колонки нет, генерится `a1…aN`. В `категория`
пустоту можно помечать словом «пусто» — трактуется как пусто.

- **лид**: `person` | `discussion` | `section` | `photo` | `photo-pair`
- **кто**: `person` → один id из «Люди»; `discussion` → 2–3 id через запятую; `section` → эмодзи (фолбэк, если нет «изображение»)
- **изображение**: `section` → фото сообщества (рисуется круглым аватаром); `photo` → URL (сквиркл `picture`); `photo-pair` → 2 URL через запятую (мини-компонент пары фото)
- **бейдж**: `онлайн` → зелёная точка на аватаре (иначе пусто)
- **текст**:
  - для `person` — **без имени**: имя само добавляется жирным из «Люди» по `кто`; род глагола — токен `{муж/жен}` (напр. `выиграл{/а}` → м «выиграл», ж «выиграла»); иконка билета — `{билет}`;
  - для остальных — свободный текст, `**жирным**` выделяешь вручную.
- **кнопка**: подпись кнопки
- **категория**: `win` | `neuro` | `holiday` | пусто — анимация подложки на входе (у `holiday` добавляется конфетти).

Картинки можно заполнять не везде — пустые `изображение` у `photo`/`photo-pair`/`section` просто дадут пустой/эмодзи-визуал.

## Как собрать

```sh
node scripts/fetch-activity.mjs
```

Скрипт: тянет лист через gviz-CSV, резолвит людей/имена/род из `data/people.json`,
рендерит ячейки и **вставляет их в три места** между маркерами:
- `new-vision/lenta.html` — виджет-конвейер `#activityConveyor` (между `ACTIVITY-WIDGET:START/END`);
- `new-vision/okruzhenie.html` — список `#activityList` (между `ACTIVITY:START/END`, промо-баннер «Копите народные билеты» сверху сохраняется);
- `activity-lenta/lenta.html` — виджет-конвейер `#activityConveyor` (компонент `activity-widget`, остров перед ВВЗ; между `ACTIVITY-WIDGET:START/END`). У этой страницы `<base href="../">`, поэтому ячейки для неё рендерятся с путями БЕЗ «../» (`cellsBase`).

Плюс пишет `data/activity.json` (выгрузка «как есть»).

Первый запуск (маркеров ещё нет) вырезает legacy-ячейки по якорям и ставит маркеры;
дальше пересборка идёт строго между маркерами.

Если таблица недоступна (HTTP 401/403) — она не опубликована «всем, у кого есть
ссылка». Можно прочитать лист через `mcp__Google_Drive__read_file_content`
(`fileId` = id таблицы), сохранить во временный CSV и скормить скрипту:
`ACTIVITY_CSV_FILE=/tmp/act.csv node scripts/fetch-activity.mjs`.

## Зависимости разметки (не трогай без надобности)

Аватарки активностей раздаёт `components/people-data.js` по `data-person-avatar="ID"`,
поэтому на обеих страницах подключены `data/people.js` + `components/people-data.js`.
Анимации категорий (трофей/нейро/конфетти) и конвейер живут в самих страницах
(around-you.css + инлайн-скрипты) — скрипт их не трогает.

## Проверка (опционально)

Прогнать агента `screenshot-testing` на `new-vision/okruzhenie.html` и на виджете
в `new-vision/lenta.html` — убедиться, что аватарки/имена подставились и ячейки
не разъехались.
