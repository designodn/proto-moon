# Посты профиля из Google-таблицы

Spreadsheet ID: `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`
Лист **«профиль»** — gid `877262163` (та же таблица, что и Q3).

Источник постов для ДВУХ секций `profile.html`:
- `data-self-posts`     — личный профиль (автор = хозяин, `data-pr-*`);
- `data-stranger-posts` — чужой профиль (автор = субъект, `data-pr-subject-*`).

## Как обновить

```sh
node scripts/fetch-profile.mjs            # тянет лист → data/profile-posts.json → вставка в profile.html
node scripts/fetch-profile.mjs --offline  # пересобрать разметку из data/profile-posts.json (без сети)
```

Скрипт: gviz-CSV → `data/profile-posts.json` (выгрузка «как есть») → рендер карточек
в ТОЧНОЙ Q3-разметке (переиспользует рендереры `scripts/fetch-q3.mjs`) → вставка
ОДНИХ И ТЕХ ЖЕ постов в обе секции `profile.html`:

- `<!-- PROFILE-POSTS:SELF:START -->` … `<!-- PROFILE-POSTS:SELF:END -->` (внутри `data-self-posts`);
- `<!-- PROFILE-POSTS:STRANGER:START -->` … `<!-- PROFILE-POSTS:STRANGER:END -->` (внутри `data-stranger-posts`).

Self- и stranger-блоки отличаются ТОЛЬКО атрибутами автора (`data-pr-*` vs
`data-pr-subject-*`) — содержимое постов идентично.

## Колонки листа (матчатся ПО ИМЕНИ заголовка — порядок столбцов можно менять)

`тип · тема · рубрика · шапка · автор · заголовок · текст · описание · фото (ссылки через запятую) · комменты · репосты · лайки · ссылка · марафон · автор коммента 1 · текст коммента 1 · автор коммента 2 · текст коммента 2`

Важно: порядок счётчиков (комменты/репосты/лайки) отличается от Q3-листа —
матчинг идёт по ИМЕНИ заголовка, не по позиции.

## Автор = хозяин профиля (колонка «автор» ПУСТАЯ)

Каждый пост профиля пишет хозяин профиля, поэтому колонку «автор» НЕ заполняем.
Шапку автора скрипт НЕ запекает из `people.json`, а подставляет runtime-атрибуты,
которые заполняет страница:
- SELF      → `data-pr-avatar` / `data-pr-name` (заполняет `user-data.js`);
- STRANGER  → `data-pr-subject-avatar` / `data-pr-subject-name` (скрипт страницы).

Авторы КОММЕНТОВ — настоящие люди: id из листа резолвятся в имя/аву из
`data/people.json` (как в fetch-q3). Держи `data/people.js`/`people.json`
актуальным (скилл `fetch-people`).

## Типы карточек (`тип`)

Поддерживаются те же типы, что и в fetch-q3 (`text · photo · photo-gallery ·
video · reshare-post …`). Для профиля типичны `photo` и `reshare-post`.
- **тема/рубрика** — крошки (breadcrumbs) над шапкой автора;
- **заголовок** — крупный заголовок (ds-title-xl);
- **текст** — тело поста (длинный → сворачиваемый toggle «ещё/скрыть»);
- **фото** — ссылки через запятую (1 → single, N → галерея);
- **комменты/репосты/лайки** — счётчики actions-bar;
- **автор/текст коммента 1·2** — ветка комментов (fc-*) под постом.

## Переиспользование fetch-q3

`scripts/fetch-q3.mjs` экспортирует рендереры/хелперы и запускает `main()` только
при прямом вызове (guard `import.meta.url === pathToFileURL(process.argv[1]).href`).
`fetch-profile.mjs` импортирует `renderPost`/`attachComments`/… и передаёт свою
шапку автора через `renderPost(p, i, { authorHeader })`. Импорт fetch-q3.mjs
БЕЗ side-effects — Q3-пайплайн при этом не трогается.

## После обновления

- Сверь, что обе пары маркеров заполнены, баланс `<article>`/`</article>` ровный,

- Ручные правки внутри маркеров затрёт следующий прогон — меняй данные в листе.
- Q3-лента не затрагивается: при необходимости проверь `git diff lenta-q3.html` (пусто).
