# Подтягивание клипов из Google-таблицы

Источник: таблица проекта, лист **«Клипы»**.
Spreadsheet ID: `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`
Вкладка «Клипы»: gid `1801786104`.

Колонки листа: `id · тип · автор · заголовок · текст · лайки · комменты · репосты`.

- **тип**: `clip` — обычный клип; `vvz-clip` — слайд «Возможно, вы знакомы»
  (вставляется в ленту на своём месте по порядку строк).
- **автор**: id человека из листа **«Люди»** — имя и аватар резолвятся на
  странице через `DS_PEOPLE` (`data/people.js` + `components/people-data.js`),
  здесь храним только id.
- **видео**: по умолчанию файл `assets/clips/<id>.mp4`. Можно добавить
  столбец «видео» с прямой ссылкой или именем файла — тогда берётся он.

Результат — два файла в репо (коммитятся, рантайм-интернет не нужен):
- `data/clips.json` — каноничные данные (+ `_readme`).
- `data/clips.js` — `window.DS_CLIPS_DATA = [...]` для плеера `klipy.html`.

## Кто это рендерит

`klipy.html` на `DOMContentLoaded` строит вертикальную ленту из
`DS_CLIPS_DATA`: клипы (`type:'clip'`) + ВВЗ-слайды (`type:'vvz-clip'`,
через компонент `components/clip-vvz.js`). Вход из ленты по тапу на клип
(`?clip=&author=&likes=&reshares=&from=`) ставит «твой» клип первым, дальше
идёт ростер из таблицы по кругу.

Видео клипов кладутся в **`assets/clips/`** с именем `<id>.mp4`
(напр. `clip-1.mp4`), вертикаль 9:16, H.264/AAC.

## Как обновить — два пути

### Путь A. Скрипт (если таблица открыта «всем, у кого есть ссылка»)

```sh
node scripts/fetch-clips.mjs
```

Скрипт тянет лист через gviz-CSV по gid, нормализует строки и пишет оба файла.
Если падает на HTTP 401/403 — таблица не опубликована, иди по пути B.

### Путь B. Через Google Drive MCP (когда скрипт не достаёт таблицу)

1. Прочитай таблицу: `mcp__Google_Drive__read_file_content` с `fileId` =
   `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`. В ответе несколько таблиц —
   нужна та, где шапка `id · тип · автор · заголовок · текст · лайки · комменты · репосты`
   (короткие id вида `clip-1`, есть строка с `vvz-clip`).
2. Для каждой строки собери объект:
   - `vvz-clip` → `{ "id", "type": "vvz-clip" }`;
   - `clip` → `{ "id", "type": "clip", "author", "likes", "comments", "reshares" }`
     (+ `title`/`text`, если заполнены; + `video`, если есть столбец «видео»).
   Порядок строк сохраняй — он задаёт порядок слайдов и место ВВЗ.
3. Перезапиши `data/clips.json` (с блоком `_readme`, не теряй его) и
   `data/clips.js` (`window.DS_CLIPS_DATA = [...]`).

## После обновления

Проверь, что все `author` из листа «Клипы» есть в `data/people.js`
(иначе имя/аватар будут пустыми) — при необходимости сначала прогони
скилл **fetch-people**.
