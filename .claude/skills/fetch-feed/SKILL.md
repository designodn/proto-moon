# Сборка ленты из Google-таблицы

Spreadsheet ID (обе ленты): `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`

Три независимых пайплайна. Определи по запросу/листу, какой нужен (можно несколько):
- **New Vision** — лист «Посты» → `new-vision/lenta.html` (см. ниже).
- **Q3** — лист «Q3-посты» → `lenta-q3.html` (см. раздел «Q3-лента» в конце).
- **Трибуна** — лист «Трибуна» (gid 803749593) → `tribune.html` (см. раздел «Трибуна» в конце).

## Типографика: неразрывные пробелы (автоматом)

Во всех трёх лентах заголовок/текст/описание/комменты прогоняются через `nbsp()`
(`scripts/fetch-feed.mjs` и `scripts/fetch-q3.mjs`): короткие предлоги/союзы из
списка `HANG_WORDS` (`а в и к о с у я во до за из ко на не об от по со то`)
приклеиваются к следующему слову неразрывным пробелом (` `, U+00A0), чтобы они
не «отваливались» в конец строки. Работает автоматически при каждом прогоне —
ничего вызывать вручную не нужно. Чтобы добавить/убрать слово — правь `HANG_WORDS`
в обоих скриптах (список одинаковый).

## New Vision

Источник: лист **«Посты»**.

Колонки листа (порядок важен, читаются по позиции):
`id · тип · автор · заголовок · текст · фото (ссылки через запятую) · лайки · комменты · репосты · тема · рубрика`

## Как обновить

```sh
node scripts/fetch-feed.mjs
```

Скрипт: тянет лист через gviz-CSV → пишет `data/feed.json` (выгрузка «как есть»),
рендерит карточки в разметке NV-компонентов и вставляет их в
`new-vision/lenta.html` между маркерами `<!-- FEED:START -->` / `<!-- FEED:END -->`.
Авторы/имена/аватары — через `data-person-*` + `people-data.js` (реестр «Люди»),
поэтому сначала держи `data/people.js` актуальным (скилл `fetch-people`).

Офлайн-реген только разметки из уже сохранённого json (без обращения к таблице):

```sh
node scripts/fetch-feed.mjs --offline
```

## Типы карточек (`тип`)

`text · photo · gallery · video · clip · article · ad · birthday · gift · questions · question · group-invite · discussion · memory · contest · stories-moments`

- **автор**: id из «Люди» (`1`, `group-3`, `my_profile`, `ad-1`…). Для группы — бейдж «Сообщество»; для рекламы (`ad-N`) — ава/имя/подпись (`subtitle`) из «Люди».
- **тема/рубрика**: две крошки (последняя активна); `тема = Дзен` рисует бейдж Дзена.
- **фото**: ссылки через запятую (галерея/конкурс — несколько).
- **stories-moments**: авторы кружков берутся из колонки «автор» (через запятую).

## Компаньон-данные (НЕ в таблице — в коде)

Вложенные куски, которых нет в плоских колонках, лежат в `scripts/fetch-feed.mjs`
в объекте `COMPANION` и **привязаны к ТИПУ карточки** (не к id поста), поэтому
перенумерация строк в листе их не ломает. Это:
- `questions` — тексты и категории карточек-вопросов;
- `discussion` — топ-комментарий + «Посмотреть N ответов»;
- `birthday` / `group-invite` — друзья (аватарки + «N друзей …»);
- `gift` — получатель/даритель (`GIFT_RESHARE`, по порядку появления подарков).

Если нужно поменять эти тексты — правь `COMPANION` в скрипте (в таблице их нет).

## После обновления

- Сверь число постов и баланс `<div>`/`<article>` в `lenta.html` (открыт/закрыт поровну), отсутствие маркеров конфликта.
- Лента генерируемая: ручные правки внутри `FEED:START/END` затрёт следующий прогон — меняй данные в листе или шаблоны в скрипте.
- Коммить только по просьбе.

## Проверка (опционально)

Прогнать агента `screenshot-testing` на `new-vision/lenta.html` — убедиться, что
карточки не разъехались и аватарки/медиа подгрузились.

# Q3-лента

Источник: лист **«Q3-посты»** (gid 1662648328) в той же таблице.
Контракт типов и колонок: `data/q3-feed.schema.json`.

Колонки листа (матчатся ПО ИМЕНИ заголовка — порядок/вставку столбцов можно менять):
`id · тип · тема · рубрика · шапка · автор · заголовок · текст · описание · фото (через запятую) · лайки · комменты · репосты · ссылка · автор коммента 1 · текст коммента 1 · автор коммента 2 · текст коммента 2`

- **тема/рубрика** — крошки (breadcrumbs) шапки для `comment-as-feed`.
- **шапка** — лейбл над клипом-карточкой («может быть интересно»).
- **автор/текст коммента 1·2** — ветка комментов под ЛЮБЫМ постом (компонент
  comment-thread, `ct-*`): рисуется только при наличии текста; автор — id из «Люди».
  Клип с комментами рендерится не full-bleed, а карточкой-островом (медиа 4:3).

## Как обновить

```sh
node scripts/fetch-q3.mjs            # тянет лист → data/q3-feed.json → вставка в lenta-q3.html
node scripts/fetch-q3.mjs --offline  # пересобрать разметку из data/q3-feed.json (без сети)
```

Скрипт: gviz-CSV → `data/q3-feed.json` (выгрузка «как есть») → рендер карточек →
вставка в `lenta-q3.html` между `<!-- FEED:START -->` / `<!-- FEED:END -->`.
Авторы резолвятся из `data/people.json` по id (`1`, `group-4`, `ad-1`, `my_profile`,
`vvz-*`), поэтому держи `data/people.js`/`people.json` актуальным (скилл `fetch-people`).

## Типы карточек (`тип`)

`text · photo · photo-gallery · video · ad · group-post · vvz-portlet · on-this-day · reshare-group · reshare-post · added-friend · shared-link · gift-received · ai-gift-received · friendversary · tagged-photo · clip · memories-clip · comment-as-feed`

- **ai-gift-received** — ИИ-подарок: как `gift-received`, но кнопка `__style-ai-gift` (тёплый градиент из ИИ-подарков NV-ленты) + подложка `#FFEFE5` бордерного блока, CTA «Создать ИИ подарок».

- **comment-as-feed** — коммент как карточка ленты: feed-header (крошки тема/рубрика
  + автор + «Комментарий к <автор оригинала>»), крупный текст коммента (`.caf-text`
  22/26 regular) = `заголовок`, превью оригинала (reshare-card: `текст`=заголовок,
  `описание`=сниппет), actions-bar + ветка ОТВЕТОВ. Автор = два id: `[комментатор,
  автор оригинала]`. Компонент `components/comment-as-feed.css`.

- **text/photo/photo-gallery/video** — один компонент feed-text, разница в `media`.
- **group-post** — feed-text с автором-сообществом (`group-*`): «Подписаться» + бейдж.
- **ad** — рекламодатель (`ad-*`), как feed-ad в NV.
- **автор**: id из листа «Люди»; несколько через запятую — для `vvz-portlet`,
  `added-friend`, `friendversary`, `gift-received` (получатель, даритель).
- **ссылка**: URL для `shared-link`.
- **shared-link превью**: колонка `описание` = `Заголовок / Подзаголовок` (делится
  по первому ` / `), либо только описание + заголовок в колонке `заголовок`. Картинка
  превью — `фото`. Если текст вписан — скрипт не дёргает страницу (не нужен авто-фетч
  og:-меты, который блокируют сайты вроде RBC).
- **токен имени**: в заголовке/тексте можно писать `<id>_name` (напр. `my_profile_name`)
  — скрипт сам подставит имя человека из `people.json` (только имя, без фамилии).

## Компаньон-данные (НЕ в листе — в коде)

Вложенные куски (subtitle ВВЗ-карточек, лайки `on-this-day`, друзья/общие

# Трибуна

Источник: лист **«Трибуна»** (gid 803749593) в той же таблице. Схема колонок и
типы карточек ПОЛНОСТЬЮ совпадают с Q3-листом, поэтому рендер общий — это тот же
`scripts/fetch-q3.mjs`, запущенный с флагом `--tribune` (см. объект `FEEDS` в
начале скрипта: меняется только источник-gid, json-выгрузка и целевой html).

## Как обновить

```sh
node scripts/fetch-q3.mjs --tribune            # тянет лист → data/tribune-feed.json → вставка в tribune.html
node scripts/fetch-q3.mjs --tribune --offline  # пересобрать разметку из data/tribune-feed.json (без сети)
```

- Колонки и типы — см. раздел «Q3-лента» выше (контракт идентичен).
- Карточки вставляются в `tribune.html` между `<!-- FEED:START … --tribune … -->` /
  `<!-- FEED:END -->`; header-остров (поиск + чипсы + вайб) и таб-бар — статика, в
  листе их нет.
- Компаньон-данные (`COMPANION` в `scripts/fetch-q3.mjs`) общие с Q3 — привязаны к
  типу карточки.
