# Справочник по листам таблицы

Подробности по каждому листу-источнику: колонки, типы карточек, компаньон-данные
(то, чего НЕТ в таблице и что лежит в коде), gid'ы и нюансы. Основной поток —
в `SKILL.md`; сюда заглядывай, когда нужно разобраться в конкретном листе или
что-то сломалось.

Spreadsheet ID (все листы): `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`

Сквозные правила:
- **Люди — первыми.** Авторы/имена/аватары резолвятся по id из листа «Люди»
  (`data/people.js` + `components/people-data.js`). Держи людей актуальными до
  остальных шагов, иначе имена/авы будут пустыми.
- **Типографика (автоматом).** В лентах заголовок/текст/описание/комменты идут
  через `nbsp()` (`fetch-feed.mjs`, `fetch-q3.mjs`): короткие предлоги/союзы из
  `HANG_WORDS` приклеиваются к следующему слову неразрывным пробелом (U+00A0).
  Менять список — в `HANG_WORDS` в обоих скриптах (он одинаковый).
- **Генерируемые блоки.** Ручные правки внутри `FEED:START/END` (и аналогичных
  маркеров) затрёт следующий прогон — меняй данные в листе или шаблоны в скриптах.

## Люди — `fetch-people.mjs` → `data/people.json` + `data/people.js`

Лист **«Люди»**: `id · Имя Фамилия · фото (прямая ссылка) · пол · возраст · город · о себе`.

- Скрипт нормализует возраст (`«70 лет»→70`) и город (`«нет»→пусто`, с заглавной),
  определяет тип медиа по `Content-Type` (`image` / `video` / `null`).
- `data/people.js` = `window.DS_PEOPLE_DATA = [...]` для подключения на страницах.
- Спец-`id` **`my_profile`** = текущий пользователь прототипа (берёт `components/user-data.js`).
- **ВВЗ-люди** — строки с id `vvz-1…vvz-N` (скрипт чинит опечатки `vvvz-3`→`vvz-3`).
  Разметка карточек по страницам — разовый `wire-vvz.mjs` (см. ниже).

Применение в разметке: `data-person-avatar="ID"` (video подменяет img на video),
`data-person-name="ID"`, `data-person-bg="ID"` (фон-блюр). Подключено в
`friends.html`, `profile.html`, `menu.html`.

## New Vision лента — `fetch-feed.mjs` → `new-vision/lenta.html` (+ `data/feed.json`)

Лист **«Посты»**. Колонки (читаются ПО ПОЗИЦИИ, порядок важен):
`id · тип · автор · заголовок · текст · фото (через запятую) · лайки · комменты · репосты · тема · рубрика`

Типы карточек: `text · photo · gallery · video · clip · article · ad · birthday ·
gift · questions · question · group-invite · discussion · memory · contest · stories-moments`.

- **автор**: id из «Люди» (`1`, `group-3`, `my_profile`, `ad-1`…). Группа → бейдж
  «Сообщество»; реклама (`ad-N`) → ава/имя/подпись (`subtitle`) из «Люди».
- **тема/рубрика**: две крошки (последняя активна); `тема = Дзен` → бейдж Дзена.
- **stories-moments**: авторы кружков — из колонки «автор» (через запятую).
- Офлайн-реген разметки из json: `node scripts/fetch-feed.mjs --offline`.

**COMPANION (в коде `fetch-feed.mjs`, привязан к ТИПУ карточки, не к id):**
`questions` (тексты/категории), `discussion` (топ-коммент + «N ответов»),
`birthday`/`group-invite` (друзья), `gift` (получатель/даритель, `GIFT_RESHARE`).

## Q3 лента — `fetch-q3.mjs` → `lenta-q3.html` (+ `data/q3-feed.json`)

Лист **«Q3-посты»** (gid `1662648328`). Контракт: `data/q3-feed.schema.json`.
Колонки матчатся ПО ИМЕНИ заголовка (порядок/вставку столбцов можно менять):
`id · тип · тема · рубрика · шапка · автор · заголовок · текст · описание ·
фото (через запятую) · лайки · комменты · репосты · ссылка ·
автор коммента 1 · текст коммента 1 · автор коммента 2 · текст коммента 2`

Типы: `text · photo · photo-gallery · video · ad · group-post · vvz-portlet ·
on-this-day · reshare-group · reshare-post · added-friend · shared-link ·
gift-received · ai-gift-received · friendversary · tagged-photo · clip ·
memories-clip · comment-as-feed`.

- **group-post**: кнопка «Подписаться» (`.button-subscribe`) ТОЛЬКО когда автор —
  сообщество (`group-*`). У авторов-людей подписки в шапке НЕТ.
- **comment-as-feed**: коммент как карточка ленты; автор = два id `[комментатор,
  автор оригинала]`; компонент `components/comment-as-feed.css`.
- **ai-gift-received**: как `gift-received`, но кнопка `__style-ai-gift` + подложка
  `#FFEFE5`, CTA «Создать ИИ подарок».
- **shared-link**: колонка `описание` = `Заголовок / Подзаголовок`; картинка — `фото`.
  Если текст вписан — авто-фетч og:-меты не дёргается.
- **токен имени**: `<id>_name` (напр. `my_profile_name`) в заголовке/тексте →
  скрипт подставит имя человека (только имя).
- Офлайн-реген: `node scripts/fetch-q3.mjs --offline`.
- Статика, которой нет в листе: баннер `ll-banner`, сторис-ряд.

**COMPANION (в коде `fetch-q3.mjs`, по ТИПУ/id):** subtitle ВВЗ-карточек, лайки
`on-this-day`, друзья/общие `added-friend`, мета `shared-link`, вложенный автор
`reshare-post`, повод/вариант `gift-received`, тег `tagged-photo`, медиа `clip`.

## Профиль — `fetch-profile.mjs` → `profile.html` (+ `data/profile-posts.json`)

Лист **«профиль»** (gid `877262163`). Те же типы/рендереры, что и Q3
(`fetch-profile.mjs` импортирует `renderPost`/`attachComments`/… из `fetch-q3.mjs`
БЕЗ side-effects — Q3-пайплайн не трогается).

Колонки матчатся ПО ИМЕНИ (порядок столбцов можно менять; **порядок счётчиков
отличается от Q3** — потому и матч по имени):
`тип · тема · рубрика · шапка · автор · заголовок · текст · описание ·
фото (через запятую) · комменты · репосты · лайки · ссылка · марафон ·
автор коммента 1 · текст коммента 1 · автор коммента 2 · текст коммента 2`

Одни и те же посты вставляются в ДВЕ секции `profile.html`:
- `<!-- PROFILE-POSTS:SELF:START -->`…`:END -->` (внутри `data-self-posts`);
- `<!-- PROFILE-POSTS:STRANGER:START -->`…`:END -->` (внутри `data-stranger-posts`).

Self/stranger отличаются ТОЛЬКО атрибутами автора (`data-pr-*` vs
`data-pr-subject-*`); содержимое идентично.

- **Колонка «автор» ПУСТАЯ** — автор поста = хозяин профиля; шапка автора НЕ
  запекается, а ставится runtime-атрибутами: SELF → `data-pr-avatar`/`data-pr-name`
  (`user-data.js`), STRANGER → `data-pr-subject-*` (скрипт страницы).
- Авторы КОММЕНТОВ — настоящие люди (id → имя/ава из `people.json`).
- Офлайн-реген: `node scripts/fetch-profile.mjs --offline`.

## Клипы — `fetch-clips.mjs` → `data/clips.json` + `data/clips.js`

Лист **«Клипы»** (gid `1801786104`). Колонки:
`id · тип · автор · заголовок · текст · лайки · комменты · репосты`.

- **тип**: `clip` — обычный; `vvz-clip` — слайд «Возможно, вы знакомы» (на своём

  клип первым, дальше ростер из таблицы по кругу (компонент `components/clip-vvz.js`).

## Активности «Вокруг вас» — `fetch-activity.mjs` → виджет + `okruzhenie.html` (+ `data/activity.json`)

Лист **«Вокруг нас»**. Колонки читаются ПО НАЗВАНИЯМ (порядок любой; ключевое
слово можно с уточнениями, напр. «изображение (если есть)»):
`лид · кто · бейдж · текст · кнопка · категория · изображение` (+ опц. `id`).
`id` необязателен (генерится `a1…aN`); «пусто» в `категория` = пусто.

- **лид**: `person` | `discussion` | `section` | `photo` | `photo-pair`.
- **кто**: `person` → один id; `discussion` → 2–3 id через запятую; `section` →
  эмодзи (фолбэк, если нет «изображение»).
- **изображение**: `section` → круглый аватар сообщества; `photo` → URL (сквиркл);
  `photo-pair` → 2 URL через запятую.
- **бейдж**: `онлайн` → зелёная точка на аватаре.
- **текст**: `person` — БЕЗ имени (имя добавляется жирным из «Люди» по `кто`; род
  глагола — токен `{муж/жен}`, напр. `выиграл{/а}`; иконка билета — `{билет}`);
  остальные — свободный текст, `**жирным**` вручную.
- **категория**: `win` | `neuro` | `holiday` | пусто — анимация подложки на входе
  (у `holiday` — конфетти).

Вставляется в ТРИ места между маркерами:
- `new-vision/lenta.html` — `#activityConveyor` (`ACTIVITY-WIDGET:START/END`);
- `new-vision/okruzhenie.html` — `#activityList` (`ACTIVITY:START/END`, промо-баннер сверху сохраняется);
- `activity-lenta/lenta.html` — `#activityConveyor` (есть `<base href="../">`,
  поэтому ячейки рендерятся с путями БЕЗ «../», `cellsBase`).

Первый запуск (маркеров нет) вырезает legacy-ячейки по якорям и ставит маркеры.

## Запасной путь: таблица недоступна (HTTP 401/403)

gviz-CSV работает, только если таблица открыта «всем, у кого есть ссылка». Если
скрипт падает на 401/403 — таблица не опубликована. Тогда:

- **Через Google Drive MCP.** Прочитай лист `mcp__Google_Drive__read_file_content`
  с `fileId` = `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y` (в ответе несколько
  markdown-таблиц — выбери по шапке нужного листа; ссылки приходят с экранированием
  `\_` `\&` — убери бэкслеши). Дальше собери JSON руками по колонкам листа (не
  теряй блок `_readme`) и регенерируй `*.js` из json.
- **Через временный CSV (для активностей):** сохрани лист в CSV и скорми скрипту:
  `ACTIVITY_CSV_FILE=/tmp/act.csv node scripts/fetch-activity.mjs`.
- Для медиа-типов определяй тип по `Content-Type`:
  `curl -s -o /dev/null -w "%{content_type}" "<url>"` → `image/*`→`image`,
  `video/*`→`video`, иначе/битая → `null` (уйдёт на заглушку).
