## Инкрементально: пропуск неизменённых листов

По умолчанию каждый скрипт сначала тянет CSV листа (дёшево) и сравнивает его
хэш с сохранённым в `data/.sync-state.json`. Если лист (и код рендера) не
менялись — пересборка пропускается целиком: ни ре-рендера, ни перекачки
картинок. Так `fetch-all` при отсутствии правок отрабатывает за секунды и даёт
пустой git diff; меняются только реально затронутые страницы.

- Детект — по хэшу CSV (Google не отдаёт время правки по отдельному листу, а
  весь файл-таблица «меняется» от любой ячейки), поэтому 1 лёгкий запрос CSV на
  лист всё же делается.
- В зависимостях листа учтены: сам скрипт, общие либы и `data/people.json` для
  лент, где имена/аватары запекаются (правка «Людей» пересобирает зависимые
  страницы). Правка кода/шаблона тоже триггерит пересборку (хэш кода).
- `data/.sync-state.json` коммитим вместе с данными — он соответствует
  запечённым страницам, и после деплоя синк сразу инкрементальный.

### Флаг `--force`

`--force` **отключает инкрементальный пропуск** — лист пересобирается, даже если
его CSV и код не менялись: лист перечитывается, картинки перекачиваются и
пересжимаются, страница перерисовывается, хэш в `data/.sync-state.json`
обновляется.

```sh
node scripts/fetch-all.mjs --force          # пересобрать ВСЕ листы с нуля
node scripts/fetch-q3.mjs --tribune --force # пересобрать один лист (Трибуна)
```

`fetch-all --force` прокидывает флаг во все шаги. Когда нужен:
- поменяли настройки сжатия/шаблон/рендер и хотите применить ко **всем** уже
  закешированным картинкам и страницам разом (правка кода и так триггерит
  пересборку через хэш, но `--force` гарантирует полный проход);
- кажется, что инкрементальный кэш «залип» (рассинхрон состояния и файлов) —
  `--force` пересоберёт начисто;
- нужно перекачать медиа, хотя текст листа не менялся (например, картинку
  заменили по тому же URL — обычный детект по CSV это не ловит).

В обычной работе (правят данные в таблице) `--force` не нужен — без него быстрее
и git diff чище.

## Медиа кешируются в репо

Картинки/видео из таблицы скачиваются в репозиторий (`scripts/lib/media-cache.mjs`),
страницы ссылаются на локальные копии (`assets/<лента>/<хэш>.webp`) — рантайм не
зависит от чужих CDN. Проверка по хэшу: не менялась — не перекачивается; менялась
— старый файл удаляется (prune), кладётся новый.

Картинки автоматически сжимаются (`sharp`): ресайз до 1600px по ширине +
перекодирование в **webp** (q80). Храним только webp — png/jpg не оставляем;
не трогаем только svg (вектор) и анимированные gif. Видео не сжимаются (нет
ffmpeg). Если `sharp` не установлен — медиа сохраняются без сжатия. Аватары
людей тоже webp, поэтому хардкод-ссылки `assets/people/<id>.webp` (не из
генерации) указывают на webp.

Реген только разметки из уже сохранённого json (без сети) — у лент:
`node scripts/fetch-feed.mjs --offline`, `... fetch-q3.mjs --offline` (+ `--tribune`/
`--activity`), `... fetch-profile.mjs --offline`. Подробности по медиа — в `sheets.md`.
