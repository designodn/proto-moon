# Подтягивание людей из Google-таблицы

Источник: таблица **«люди»**, лист **«Люди»**.
Spreadsheet ID: `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y`
Лист «Люди»: `id · Имя Фамилия · фото (прямая ссылка) · пол · возраст · город · о себе`.

Результат работы — два файла в репо (коммитятся, рантайм-интернет не нужен):
- `data/people.json` — каноничные данные (+ `_readme`).
- `data/people.js` — `window.DS_PEOPLE_DATA = [...]` для подключения на страницах.

Спец-`id` **`my_profile`** = текущий пользователь прототипа (его берёт `components/user-data.js`).

## Как обновить — два пути

### Путь A. Скрипт (если таблица открыта «всем, у кого есть ссылка»)

```sh
node scripts/fetch-people.mjs
```

Скрипт сам: тянет лист через gviz-CSV, нормализует возраст (`«70 лет»→70`) и город
(`«нет»→пусто`, с заглавной), определяет тип медиа по `Content-Type`
(`image` / `video` / `null`), пишет оба файла. Если падает на HTTP 401/403 —
таблица не опубликована, иди по пути B.

### Путь B. Через Google Drive MCP (когда скрипт не достаёт таблицу)

1. Прочитай лист: `mcp__Google_Drive__read_file_content` с `fileId` =
   `1Ctwjp2J0HSmvb6kL4NoDqaB9W4QfdAXXDnzyBDLYZ7Y` (бери первую markdown-таблицу —
   это лист «Люди»; ссылки приходят с экранированием `\_` `\&` — убери бэкслеши).
2. Для каждой ссылки на фото определи тип: `curl -s -o /dev/null -w "%{content_type}" "<url>"`.
   `image/*`→`image`, `video/*`→`video`, иначе/битая/заглушка (`https://...jpg`)→`photo:null, media:null`.
3. Нормализуй: возраст → число, город `«нет»`/пусто → `""`, имя/«о себе» — как в таблице.
4. Перезапиши `data/people.json` (поля: `id, name, photo, media, gender, age, city, bio`,
   плюс блок `_readme` как в текущем файле — не теряй его).
5. Регенерируй `data/people.js` из json:
   ```sh
   node -e "const fs=require('fs');const j=JSON.parse(fs.readFileSync('data/people.json','utf8'));fs.writeFileSync('data/people.js','/* Сгенерировано scripts/fetch-people.mjs — НЕ редактировать вручную. */\n'+'window.DS_PEOPLE_DATA = '+JSON.stringify(j.people,null,2)+';\n');"
   ```

## После обновления

- Сверь количество и `id` записей (вкл. `my_profile`).
- Если менялись «живые фото» (video) или появились битые ссылки — упомяни это
  пользователю: video рендерится как `<video>`, `media:null` уходит на заглушку.
- Покажи краткий дифф (кто добавился/сменил фото). Коммить только по просьбе.

## Как это используется в разметке (не трогай без надобности)

Подключение: `<script src="data/people.js"></script>` затем
`<script src="components/people-data.js"></script>`. Аппликатор раздаёт людей по
`data-person-avatar="ID"` (img→при video подменяется на video), `data-person-name="ID"`,
`data-person-bg="ID"` (фон-блюр). Текущий пользователь — `my_profile` через `user-data.js`.
Подключённые примеры: `friends.html`, `profile.html`, `menu.html`.

## ВВЗ-блоки «Возможно, вы знакомы»

Люди для свайп-карточек ВВЗ — это строки листа «Люди» с id `vvz-1`…`vvz-N`
(скрипт чинит опечатки вида `vvvz-3` → `vvz-3`). Разметка карточек по всем
страницам — разовым трансформером `node scripts/wire-vvz.mjs` (проставляет
`data-person-*` на `.vvz-card` / `.friend-card`, циклично если карточек больше,
чем людей; блок `dnoBlock` «Давно не общались» не трогает). Запускать после
добавления/изменения vvz-людей, если появились новые ВВЗ-блоки в разметке.

## Проверка (опционально)

Чтобы убедиться, что аватарки реально грузятся в браузере, запусти агента
`screenshot-testing` на нужной странице (например `friends.html`).
