# Railway: хостинг прототипа + автообновление ленты из Google-таблицы

> ⚠️ **Не основной путь.** Прототип переехал на **Yandex Serverless Containers**
> — см. `YANDEX-CLOUD.md`. Там тот же `server.mjs` с живой кнопкой «Обновить
> ленту». Этот файл оставлен как справка по прежнему хостингу на Railway.


В отличие от чисто статического деплоя (см. `DEPLOY.md`), здесь прототип
раздаёт маленький Node-сервер `server.mjs`, который заодно сам обновляет ленту
из Google-таблицы. Бэкенд-логики у прототипа по-прежнему нет — сервер только
отдаёт файлы и пересобирает данные при старте и по кнопке.

## Как это работает

1. `server.mjs` слушает порт из `PORT` (Railway задаёт сам) и раздаёт все файлы
   репозитория как статику.
2. При старте сервера и по кнопке «Обновить ленту» на лендинге он запускает
   `scripts/fetch-all.mjs`. Тот читает **публичную** Google-таблицу
   (gviz-CSV), перезаписывает `data/*.json` и вставляет свежие карточки в
   страницы (`new-vision/lenta.html`, `lenta-q3.html`, `tribune.html` и т.д.).
3. Браузер при следующем заходе получает уже обновлённые файлы.

Изменения остаются только в контейнере и **не коммитятся в git**. После
рестарта/редеплоя Railway данные пересобираются заново из таблицы.

> Обязательное условие: таблица открыта **«всем, у кого есть ссылка»**, иначе
> gviz-CSV не отдаст данные и синк будет падать (сайт при этом продолжит
> раздаваться со старыми данными).

## Пути прототипов

Сервер раздаёт несколько прототипов с разных адресов:

| Путь          | Открывает                          |
|---------------|------------------------------------|
| `/`           | разводящая страница: выбор прототипа + кнопка «Обновить ленту» |
| `/nv`         | New Vision — `new-vision/lenta.html` |
| `/activity`   | «Вокруг вас» — `new-vision/okruzhenie.html` |
| `/q3`         | Q3 — `lenta-q3.html`               |
| `/preview`    | витрина дизайн-системы — `preview.html` |
| `/healthz`    | статус сервера и последнего синка (`{"ok":true,"syncing":…,"lastSync":…}`) |
| `POST /api/sync` | запустить синк вручную (его дёргает кнопка на лендинге) |

Красивые пути отвечают редиректом на реальную страницу — так относительные
ссылки на стили и ассеты внутри страницы продолжают работать.

### Кнопка «Обновить ленту из таблицы»

На разводящей странице есть кнопка ручного обновления. По клику она шлёт
`POST /api/sync` (запускает `fetch-all.mjs` на сервере), затем опрашивает
`/healthz` и показывает статус: «Обновляю…», «Готово — обновлено и запушено в
HH:MM» или ошибку. Параллельный запуск защищён — пока синк идёт, повторный
игнорируется.

После успешного синка сервер **коммитит изменения и пушит** их в git. Для пуша
нужен токен с правом записи (`GITHUB_TOKEN`); без него коммит сделается, но пуш
не пройдёт. Полный список переменных авто-коммита (`GITHUB_TOKEN`,
`SYNC_GIT_PUSH_URL`, `SYNC_GIT_BRANCH`, `SYNC_GIT_COMMIT`, `SYNC_GIT_NAME`/
`SYNC_GIT_EMAIL`, `GIT_REPO_SLUG`) — в **`YANDEX-CLOUD.md`** (раздел «Авто-коммит
результата синка»), он же актуальный путь деплоя.

## Настройка на Railway

1. **New Project → Deploy from GitHub repo**, выберите этот репозиторий и ветку.
2. Railway увидит `package.json` и соберёт Node-проект (Nixpacks).
   Команда запуска по умолчанию — `npm start` (то есть `node server.mjs`).
   Зависимость одна — `sharp` (сжатие картинок при синке); `npm install`
   ставит её из prebuilt-бинарника, сам сервер её не требует.
3. В **Settings → Networking** включите публичный домен (Generate Domain).
4. В **Variables** задайте:
   - `GITHUB_TOKEN` — токен с правом записи в репо (чтобы кнопка пушила синк;
     без него пуш не пройдёт). См. «Кнопка…» выше про остальные `SYNC_GIT_*`;
   - `SYNC_ON_START` — `false`, чтобы не синкать при старте сервера
     (тогда лента обновляется только по кнопке);
   - `PORT` задавать не нужно — Railway проставляет его сам.

Готово: по выданному домену откроется лендинг, по `/nv`, `/q3`, `/activity` —
конкретные прототипы, а лента будет подтягиваться из таблицы автоматически.

## Локальная проверка

```bash
npm start                      # http://localhost:3000 (синк при старте)
# без обращения к таблице на старте:
SYNC_ON_START=false npm start
# разовый ручной синк из консоли:
npm run sync
```
